# 总体架构设计说明书

## 架构原则

采用模块化单体业务后端、插件化 AI 流水线和独立模型网关。模块通过公开接口协作，禁止跨租户访问和由大模型生成订单事实。

```text
Web 工作台 → FastAPI/API 鉴权 → 会话编排
                         ├→ 电商业务事实解析 → OMS/WMS 适配器
                         ├→ 知识检索 → 证据门控 → 模型网关
                         └→ 拒答/转人工 → 工单中心
数据层：PostgreSQL/SQLite；缓存与限流：Redis；观测目标：OpenTelemetry
```

## 模块

| 模块 | 职责 |
|---|---|
| auth/audit | 登录、JWT、RBAC、租户校验、操作留痕 |
| customer | 客户档案、会员和标签 |
| commerce | 订单、物流、退款事实适配 |
| conversation | 会话、消息和问答入口 |
| knowledge | 企业知识与检索 |
| handoff | 人工转接与工单流转 |
| analytics | 运营指标聚合 |
| ai_platform | 缓存、检索、生成、模型路由和降级 |

## AI 决策顺序

业务接口 → 标准缓存 → 知识检索 → 证据阈值 → 模型生成 → 引用返回。证据不足时拒答；模型失败时返回原始证据。当前检索为轻量词元匹配，生产应替换为混合检索与重排。

## 扩展方式

新功能建立独立 `modules/<name>`、`module.yaml`、路由、服务、表和测试，通过 `ModuleRegistry` 装配；使用 `FEATURE_*` 开关启停。
