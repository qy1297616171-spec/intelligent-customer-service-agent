# 需求分析说明书

## 建设目标

建设面向电商售前、物流、退款、退换货、发票等场景的企业智能客服平台，实现业务事实直查、知识问答、低置信度拒答、转人工闭环及运营分析。

## 用户角色

- 企业所有者/管理员：成员、权限、审计及全部业务管理。
- 内容编辑：知识库和客户标签维护。
- 客服人员：会话处理、订单查询、转人工和工单流转。
- 只读成员：查看数据，不可变更。
- 商城客户：通过客服入口咨询。

## 功能需求

1. 会话：新建、历史消息、搜索、全部/待处理筛选。
2. 电商业务：订单、物流、退款状态确定性查询，不允许模型推断。
3. 知识库：新增、查询、编辑、删除、引用及租户隔离。
4. AI：证据门控、模型网关、缓存、超时降级、引用返回。
5. 工单：转人工、优先级、排队、处理、解决和关闭。
6. 客户：会员等级、标签、来源、地区、订单和消费画像。
7. 分析：会话、消息、AI 解决率、回答类型和工单趋势。
8. 安全：登录、JWT Cookie、RBAC、租户校验和审计日志。

## 非功能需求

- 简单缓存请求目标响应小于 300ms；RAG 首响应目标 2s 内。
- 动态业务数据必须来自 OMS/ERP/WMS；敏感数据最小化展示。
- 单模块可启停、可替换；日志不得记录密码或 API Key。
- 数据需备份、可恢复、可审计；生产环境必须 HTTPS。

## 验收口径

核心接口成功、越权被拒绝、无证据拒答、工单闭环、重启数据保留、测试全通过。真实模型、真实订单系统、压测和灾备演练属于生产上线前置验收。
