# 数据库设计说明书

生产推荐 PostgreSQL 16；本地默认 SQLite。字符集统一 UTF-8，时间当前保存 ISO 8601 UTC 字符串。

| 表 | 主键 | 关键字段 | 用途 |
|---|---|---|---|
| tenants | id | name,status | 企业租户 |
| users | id | tenant_id,email,role,password_hash | 企业成员 |
| customers | id | tenant_id,membership,tags,total_spent | 客户画像 |
| knowledge_documents | id | tenant_id,title,content,source | 知识库 |
| knowledge_embeddings | document_id | tenant_id,model,vector | 知识向量 |
| knowledge_vectors_pg | document_id | tenant_id,embedding vector(256) | PostgreSQL HNSW 向量索引 |
| conversations | id | tenant_id,customer_id,status | 会话 |
| conversation_messages | id | conversation_id,role,citations | 消息 |
| handoff_tickets | id | tenant_id,conversation_id,status,priority | 工单 |
| audit_logs | id | tenant_id,user_id,action,resource | 审计 |

租户表均应以 `tenant_id` 作为查询条件；消息通过 `conversation_id` 关联会话；工单关联会话。用户密码只保存 Argon2 哈希。引用字段和标签使用 JSON。

数据库已升级至 Alembic `0003_pgvector`，部署执行 `alembic upgrade head`。PostgreSQL 会启用 vector 扩展及余弦 HNSW 索引，SQLite 跳过该物理表。应用保留 `create_all` 作为开发兼容保护，后续结构变更必须新增迁移版本。数据库账号应分权并启用连接加密、慢查询和备份。
