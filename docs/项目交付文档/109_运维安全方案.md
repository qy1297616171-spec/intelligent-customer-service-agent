# 运维安全方案

## 日常运维

监控可用率、P95/P99 延迟、5xx、数据库连接、拒答率、转人工率、模型超时和 Token 成本。告警分 P1/P2/P3；P1 立即响应并启动应急群。

## 安全控制

- 全站 HTTPS，Cookie 使用 HttpOnly、Secure、SameSite。
- owner/admin 最小化，季度复核权限，离职即时回收。
- 密钥进入 KMS/Secret，不写代码、日志或镜像。
- 数据库最小权限、网络白名单、备份加密。
- 依赖和镜像定期扫描，严重漏洞限定时修复。
- WAF、限流、防爆破；登录失败异常告警。
- 输入防提示词注入；模型只能读取授权证据；输出检测隐私。

## 事件响应

发现异常后隔离入口/账号→保全日志→评估影响租户→修复或回滚→通知相关方→复盘。疑似密钥泄露必须立即轮换；疑似数据泄露不得删除现场日志。

变更执行申请、评审、备份、灰度、验证、关闭流程。生产操作使用实名账号，禁止共享管理员；审计日志定期归档并校验完整性。
# 2026-06-30 安全运行补充

系统已默认启用登录、Argon2 密码哈希、JWT HttpOnly Cookie、五级 RBAC、租户边界校验、登录/API 限流和关键写操作审计。生产环境必须使用 HTTPS、设置 `AUTH_COOKIE_SECURE=true`、替换演示账号和全部默认密码，并通过密钥管理服务注入模型、OMS、数据库及 Grafana 凭据。

Prometheus `/metrics` 当前为便于内网抓取而免登录，公网部署时必须由防火墙或反向代理限制为监控网络可访问。Grafana 不得使用模板默认密码。发布前执行依赖漏洞扫描、镜像扫描、越权回归、提示词注入、敏感信息泄漏和审计日志完整性检查。
